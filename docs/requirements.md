# Argus 需求文档

## 1. 项目概述

### 1.1 背景
核电站关键区域（反应堆厂房、安全壳内、管廊、乏燃料水池周边等）实行严格的异物防控（Foreign Object Exclusion, FOE）。任何非预期物体（墙皮脱落、保温层掉落、遗留工具、杂物等）都可能对设备安全构成威胁。

### 1.2 目标
通过固定摄像头 + 深度学习异常检测，实现：
- 实时监控关键区域，自动发现异常物体
- 分级报警，推送到控制室
- 最小化误报率（核电站环境下报警必须有意义）
- 7x24 小时不间断运行

### 1.3 系统名称
Argus（阿耳戈斯，希腊神话中的百眼巨人）

## 2. 功能需求

### 2.1 视觉检测管线

#### FR-001 摄像头采集
- 支持 RTSP、USB、视频文件三种输入源
- 支持配置目标帧率（帧率抽取，默认 5FPS）
- 支持配置分辨率
- 断线自动重连（非阻��），指数退避，可配置最大重试次数

#### FR-002 预筛阶段（MOG2 背景减除）
- 使用 MOG2 算法建模背景，检测前景变化
- 无变化的帧直接跳过，节约后续算力
- 阴影检测开关（避免阴影触发假��性）
- 变化比例阈值可配置
- **辐射环境适配**：前置中值滤波消除 CMOS 辐射椒盐噪声
- **相位相关图像对齐**：补偿摄像头微震动（<5px）

#### FR-003 人员过滤（YOLO）
- 使用 YOLO11n 多类检测（80 类 COCO），支持 BoT-SORT 目标追踪
- 两种模式：遮罩（高斯模糊覆盖人员区域）或跳帧（整帧跳过）
- YOLO 不可用时优雅降级（跳过人员过滤，不崩溃）
- 多摄像头共享单个 YOLO 实例

#### FR-004 异常检测（Anomalib）
- 支持 PatchCore、EfficientAD、FastFlow、PaDiM 四种算法
- 只需要"正常"图像训练，不需要异常样本标注
- 输出：异常分数（0-1）+ 逐像素异常热力图
- 冷启动回退：无模型时使用 SSIM 占位
- **模型热更新**：运行中原子替换模型，不中断推理
- **多模型集成**：多个模型投票/加权融合

#### FR-005 防背景吸收
- **心跳全帧检测**：每 N 帧强制跳过 MOG2，直接送检测
- **异常状态锁定**：高置��度异常触发锁定，冻结 MOG2 学习率

### 2.2 区域管理

#### FR-006 检测区域（Include Zone）
- 多边形定义，限定检测范围
- 多个检测区域可共存，各自独立评估和报警
- 每个区域可设置独立的异常阈值和优先级

#### FR-007 排除区域（Exclude Zone）
- 多边形定义，该区域内的任何变化都不检测
- 排除区域优先级高于检测区域

#### FR-008 区域热更新
- 区域增删不需要重启系统

### 2.3 报警系统

#### FR-009 四级报警分级
- INFO / LOW / MEDIUM / HIGH
- 区域优先级乘数（CRITICAL 1.2x / STANDARD 1.0x / LOW_PRIORITY 0.8x）

#### FR-010 时间窗滤波
- 连续 N 帧确认 + 空间 IoU 连续性检查
- 自适应阈值（基于滑动窗口分数分布）

#### FR-011 重复抑制
- 同一区域/同一摄像头时间窗内不重复报警

#### FR-012 报警分发
- 数据库持久化 + Webhook + 邮件 + WebSocket 实时推送
- 分发断路器（三态：closed/open/half-open）
- 关闭时 drain 队列，防止数据丢失

#### FR-013 报警反馈
- 操作员确认 / 标记误报
- 告警工作流状态机（NEW → ACKNOWLEDGED → INVESTIGATING → RESOLVED → CLOSED）
- 浏览器音频告警（3 个严重等级）

### 2.4 多摄像头支持

#### FR-014 线程池管理
- 每个摄像头运行在独立线程中
- 单个摄像头故障不影响其他摄像头
- 支持运行时独立启停

### 2.5 Dashboard 仪表盘

#### FR-015 ~ FR-020 核心页面
- 系统概览（状态卡片、摄像头网格、最近告警）
- 摄像头管理（添加/启停/统计/MJPEG 流）
- 告警中心（筛选/批量操作/CSV 导出/工作流状态）
- 区域编辑器（Canvas 多边形绘制）
- 检测调试视图（帧级诊断/灵敏度预览/学习模式）
- 系统设置（阈值调整/锁定清除/模型热更新）
- JSON API（health/alerts/cameras）

#### FR-015a ~ FR-015d 运维页面
- 审计日志（操作追踪、用户/动作/目标）
- 用户管理（RBAC：admin/operator/viewer）
- 统计报表（日报/周报、误报率趋势）
- 数据备份/恢复（SQLite 在线备份）

### 2.6 基线与训练

#### FR-021 基线管理
- 版本化存储（v001, v002...），旧版本自动清理
- 采集质量过滤（模糊/过曝/重复/编码错误/有人帧自动排除）
- 多时段采集提示（白天/夜间/检修覆盖性检查）

#### FR-022 模型训练
- 训练前验证（数量/损坏率/重复率/亮度检查）
- 自动切分验证集（80/20, seed=42）
- 训练后质量报告（A/B/C/F 等级 + 分数分布 + 推荐阈值）
- 训练历史记录（TrainingRecord ORM）
- 模型 A/B ��比
- 导出到 OpenVINO / ONNX 格式

#### FR-023 误报反馈闭环
- 误报快照自动导出到基线目录
- 下次重训练时纳入

### 2.7 部署

#### FR-024 容器化
- Docker 多阶段构建
- docker-compose 一键部署
- 数据卷持久化、资源限制

## 3. 非功能需求

### NFR-001 延迟
- 从帧采集到报警触发 < 500ms

### NFR-002 可用性
- 7x24 小时不间断运行
- 单摄像头故障不影响全局
- 自动重连、自动恢复

### NFR-003 误报率
- 时间窗滤波 + 区域掩码 + 辐射去噪 + 自适应阈值，目标 < 1 次/天/摄像头

### NFR-004 安全
- Docker 非 root 用户运行
- RBAC 用户权限（admin/operator/viewer）
- 操作审计日志

### NFR-005 可维护性
- Pydantic 类型安全配置
- ���构化日志（structlog）
- 322+ 单元测试

## 4. 约束条件

- 边缘部署：Intel NUC 或 NVIDIA Jetson Orin NX
- 每节点 2-4 路摄像头
- Python 3.11+
- 核电站网络环境（可能无外网访问）
