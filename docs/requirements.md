# Argus 需求文档

## 1. 项目概述

### 1.1 背景
工业场所、基础设施、仓储园区等关键区域需要持续的环境监控。任何非预期物体（遗留工具、结构损伤、杂物堆积等）或场景变化都可能影响安全生产与运营效率。传统人工巡检存在覆盖不足、响应滞后、主观判断差异大等问题。

### 1.2 目标
通过固定摄像头 + 深度学习异常检测，实现：
- 实时监控关键区域，自动发现异常物体与场景变化
- 四级分级报警（INFO/LOW/MEDIUM/HIGH），多通道推送到值班台
- 最小化误报率（目标 < 1 次/天/摄像头），报警必须有意义
- 7×24 小时不间断运行，支持边缘部署与内网离线环境

### 1.3 系统名称
Argus（阿耳戈斯，希腊神话中的百眼巨人）

## 2. 功能需求

### 2.1 视觉检测管线

#### FR-001 摄像头采集
- 支持 RTSP、USB、视频文件三种输入源
- 支持配置目标帧率（帧率抽取，默认 5FPS）
- 支持配置分辨率
- 断线自动重连（非阻塞），指数退避，可配置最大重试次数

#### FR-002 预筛阶段（MOG2 背景减除）
- 使用 MOG2 算法建模背景，检测前景变化
- 无变化的帧直接跳过，节约后续算力
- 阴影检测开关（避免阴影触发假阳性）
- 变化比例阈值可配置
- **噪声适配**：前置中值滤波消除 CMOS 椒盐噪声（适用于高干扰环境）
- **相位相关图像对齐**：补偿摄像头微震动（<5px）

#### FR-003 人员过滤（YOLO）
- 使用 YOLO11n 多类检测（80 类 COCO），支持 BoT-SORT 目标追踪
- 两种模式：遮罩（高斯模糊覆盖人员区域）或跳帧（整帧跳过）
- YOLO 不可用时优雅降级（跳过人员过滤，不崩溃）
- 多摄像头共享单个 YOLO 实例

#### FR-004 异常检测（Anomalib）
- 支持 PatchCore、EfficientAD、FastFlow、PaDiM、**Dinomaly2**（CVPR 2025 SOTA，DINOv2 backbone）五种算法
- 只需要"正常"图像训练，不需要异常样本标注
- **Few-shot 模式**：Dinomaly2 最少 8 张图像即可训练
- **多类共享模型**：单个 Dinomaly2 模型可跨摄像头共享，支持多类别场景
- 输出：异常分数（0-1）+ 逐像素异常热力图
- 冷启动回退：无模型时使用 SSIM 占位
- **模型热更新**：运行中原子替换模型，不中断推理
- **多模型集成**：多个模型投票/加权融合
- **INT8 量化导出**：NNCF PTQ 量化，降低推理延迟
- **Conformal Prediction 分数校准**：分布无关的 FPR 保证，运行时校准异常阈值

#### FR-005 防背景吸收
- **心跳全帧检测**：每 N 帧强制跳过 MOG2，直接送检测
- **异常状态锁定**：高置信度异常触发锁定，冻结 MOG2 学习率

#### FR-005a Simplex 双通道安全架构
- **Simplex 安全通道**：纯 OpenCV 帧差分检测器，与 ML 通道并行运行
- 双通道同时确认可提升置信度
- ML 通道失效时，Simplex 提供安全兜底，保障检测不中断

### 2.2 区域管理

#### FR-006 检测区域（Include Zone）
- 多边形定义，限定检测范围
- 多个检测区域可共存，各自独立评估和报警
- 每个区域可设置独立的异常阈值和优先级

#### FR-007 排除区域（Exclude Zone）
- 多边形定义，该区域内的任何变化都不检测
- 排除区域优先级高于检测区域

#### FR-008 区域热更新
- 区域增删不需要重启系统

### 2.3 报警系统

#### FR-009 四级报警分级
- INFO / LOW / MEDIUM / HIGH
- 区域优先级乘数（CRITICAL 1.2x / STANDARD 1.0x / LOW_PRIORITY 0.8x）

#### FR-010 CUSUM 时序证据累积
- CUSUM 指数证据累积（λ=0.95, threshold=3.0），替代硬性连续帧计数
- 软证据衰减：异常帧累加证据，正常帧指数衰减，超过阈值触发报警
- 空间 IoU 连续性检查
- 自适应阈值（基于滑动窗口分数分布）

#### FR-011 重复抑制
- 同一区域/同一摄像头时间窗内不重复报警

#### FR-012 报警分发
- 数据库持久化 + Webhook + 邮件 + WebSocket 实时推送
- 分发断路器（三态：closed/open/half-open）
- 关闭时 drain 队列，防止数据丢失

#### FR-013 报警反馈
- 操作员确认 / 标记误报
- 告警工作流状态机（NEW → ACKNOWLEDGED → INVESTIGATING → RESOLVED → CLOSED）
- 浏览器音频告警（3 个严重等级）

### 2.4 多摄像头支持

#### FR-014 线程池管理
- 每个摄像头运行在独立线程中
- 单个摄像头故障不影响其他摄像头
- 支持运行时独立启停

### 2.5 Dashboard 仪表盘

#### FR-015 ~ FR-020 核心页面
- **前端架构**：Vue 3 + Ant Design Vue SPA（暗色主题），WebSocket 实时更新（health/cameras/alerts/tasks 四个主题）
- 系统概览（状态卡片、摄像头网格、最近告警）
- 摄像头管理（添加/启停/统计/go2rtc 视频流 + MJPEG 回退）
- 告警中心（筛选/批量操作/CSV 导出/工作流状态）
- **告警回放取证**（多轨时间线/逐帧检视/信号历史/参考帧对比）
- 区域编辑器（Canvas 多边形绘制）
- 检测调试视图（帧级诊断/灵敏度预览/学习模式）
- 系统设置（阈值调整/锁定清除/模型热更新）
- **模型管理页**：10 个子组件（骨干网络管理/基线管理/训练创建/训练历史/训练作业列表/模型表格/模型标签页/批量推理/事件日志/训练标签页）
- **降级监控**（全局降级条 + 降级事件历史 + 影响评估）
- **训练作业管理**（创建/确认/拒绝/进度跟踪/骨干网络升级）
- **视频流管理**（go2rtc 代理状态与连接注册）
- JSON API（health/alerts/cameras），用户管理与审计日志使用 JSON API

#### FR-015a ~ FR-015d 运维页面
- 审计日志（操作追踪、用户/动作/目标）
- 用户管理（RBAC：admin/operator/viewer）
- 统计报表（日报/周报、误报率趋势）
- 数据备份/恢复（SQLite 在线备份）

### 2.6 基线与训练

#### FR-021 基线管理
- 版本化存储（v001, v002...），旧版本自动清理
- 采集质量过滤（模糊/过曝/重复/编码错误/有人帧自动排除）
- 多时段采集提示（白天/夜间/检修覆盖性检查）

#### FR-022 模型训练
- 训练前验证（数量/损坏率/重复率/亮度检查）
- 自动切分验证集（80/20, seed=42）
- 训练后质量报告（A/B/C/F 等级 + 分数分布 + 推荐阈值）
- 训练历史记录（TrainingRecord ORM）
- 模型 A/B 对比
- 导出到 OpenVINO / ONNX 格式

#### FR-023 误报反馈闭环
- 误报快照自动导出到基线目录
- 下次重训练时纳入

### 2.7 部署

#### FR-024 容器化
- Docker 多阶段构建
- docker-compose 一键部署
- 数据卷持久化、资源限制

### 2.8 摄像头健康与漂移监控

#### FR-025 摄像头健康监控
- 检测五种异常状态：画面冻结、镜头污染、摄像头位移、闪光干扰、图像漂移
- 异常状态自动上报，触发操作员告警

#### FR-026 KS 漂移监控
- 参考窗口 → 滑动窗口 KS 检验
- 分数分布偏移超过阈值时生成操作员告警

### 2.9 跨摄像头与高级检测

#### FR-027 跨摄像头异常关联
- 单应性投影（Homography Projection），将不同视角映射到统一坐标系
- 跨摄像头佐证检查：多视角同时检测到异常可提升置信度

#### FR-028 开放词汇检测
- YOLO-World + 自定义词汇表，识别具体异常物体类别
- 基于风险的严重程度调整（不同类别异物映射不同报警等级）

#### FR-029 SAM 2 实例分割
- Point-prompted 分割：基于异常检测定位点，自动分割异常区域轮廓
- 为告警提供精确的异常区域 mask

#### FR-030 合成异常数据生成
- 合成管线：将异常物体合成到正常背景图像上
- 用于召回率评估与模型验证

### 2.10 模型管理与主动学习

#### FR-031 模型版本管理
- ModelRegistry：注册/激活/回滚模型版本
- 告警记录携带 model_version_id，支持版本溯源
- **模型发布管线**：shadow（≥3 天影子推理）→ canary（≥7 天金丝雀）→ production 三阶段晋升
- 影子推理日志对比（ShadowInferenceLog），自动计算新旧模型分数差异
- 模型阶段变更事件记录（ModelVersionEvent）

#### FR-032 主动学习基线优化
- 多样性选择：从采集图像中选取最具代表性的子集
- FP 反馈闭环：误报标注自动纳入下一轮训练

### 2.11 视频流、回放与系统监控

#### FR-033 go2rtc 视频流代理
- 集成 go2rtc 子进程，提供 WebRTC/MSE/HLS/MJPEG 多格式视频流
- 端口：1984（API）/ 8554（RTSP）/ 8555（WebRTC）
- 动态注册摄像头流，支持 RTSP 直通
- MJPEG 作为回退方案

#### FR-034 告警回放取证
- **环形缓冲区**：前触发 60s + 后触发 30s 帧录制（AlertRingBuffer）
- 逐帧回放：支持异常帧序列、热力图序列检视
- 多轨信号可视化（Dinomaly 分数、Simplex 分数、CUSUM 证据）
- 参考基线帧对比
- 关键帧固定标记（pin-frame）

#### FR-035 降级检测与追踪
- 检测五种降级场景：画面冻结、镜头污染、摄像头位移、闪光干扰、模型漂移
- 降级状态机：正常 → 警告 → 降级 → 恢复
- 降级事件历史记录与影响评估
- 全局降级状态条（DegradationBar 组件）

#### FR-036 骨干网络 SSL 训练
- DINOv2 骨干网络共享微调（backbone_trainer）
- 骨干网络生命周期管理（backbone_manager）：注册/升级/状态追踪
- 骨干升级后自动触发下游模型重训练

#### FR-037 批量推理
- 上传图像集批量推理，返回异常分数与热力图
- 用于离线验证模型效果

#### FR-038 基线生命周期管理
- 基线版本化存储（BaselineVersionRecord）
- 版本状态：active / retired
- 基线激活/退役 API
- 基线组合并（merge groups）
- 误报帧自动合并到基线（merge-fp）

#### FR-039 推理记录与缓存
- 逐帧推理元数据记录（InferenceRecord ORM）
- 内存推理结果缓存（InferenceBuffer），减少重复查询
- 推理结果持久化（InferenceStore）

#### FR-040 智能帧采样策略
- 三种采样模式：多样性采样/主动采样/定时采样
- 基于特征向量的多样性选择，最大化训练集覆盖度

#### FR-041 数据契约验证
- 管线边界输入/输出数据契约（contracts/validation.py）
- 确保模块间数据格式一致性

#### FR-042 训练作业管理
- 训练作业生命周期 API（创建/确认/拒绝/暂停/恢复/终止）
- 作业状态持久化（TrainingJobRecord ORM）
- 骨干网络可用列表查询
- 异步作业执行器（job_executor）

## 3. 非功能需求

### NFR-001 延迟
- 从帧采集到报警触发 < 500ms

### NFR-002 可用性
- 7x24 小时不间断运行
- 单摄像头故障不影响全局
- 自动重连、自动恢复

### NFR-003 误报率
- 时间窗滤波 + 区域掩码 + 噪声抑制 + 自适应阈值，目标 < 1 次/天/摄像头

### NFR-004 安全
- Docker 非 root 用户运行
- RBAC 用户权限（admin/operator/viewer）
- 操作审计日志

### NFR-005 可维护性
- Pydantic 类型安全配置
- 结构化日志（structlog）
- 935 测试函数（单元 + 集成 + 根级测试，覆盖 72 个测试文件）

## 4. 约束条件

- 边缘部署：Intel NUC 或 NVIDIA Jetson Orin NX
- 每节点 2-4 路摄像头
- Python 3.11+
- PyTorch 2.11.0+cpu（无 CUDA/GPU）
- OpenVINO 2026.0.0
- 内网离线环境（可能无外网访问）
