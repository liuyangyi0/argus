# Argus 当前需求与范围说明

> 最后更新：2026-04-11

本文档描述当前仓库的产品目标、已实现范围和近期约束，用来替代早期多轮迭代后已经失真的“全量愿景列表”。

## 1. 产品目标

Argus 的当前目标是：

- 使用固定摄像头持续检测场景异常或异物。
- 在资源受限的单节点环境中稳定运行。
- 为操作员提供可追溯的告警、回放和处理流程。
- 支持基线采集、模型训练、模型切换与问题复盘。

## 2. 当前已实现的核心需求

### 2.1 视频采集与运行管理

- 支持 RTSP、USB、文件输入。
- 支持按摄像头独立运行、重连和状态查询。
- 支持基础画质检查与摄像头健康相关能力。

### 2.2 检测管线

- 支持 include / exclude 区域限制。
- 支持 MOG2 预筛与 heartbeat 旁路。
- 支持基于 YOLO 的人员过滤。
- 支持异常检测主通道。
- 支持 Simplex 安全通道并行判断。
- 支持后处理、时序证据累积、自适应阈值与告警分级。

### 2.3 告警与处置

- 告警入库并支持筛选查询。
- 支持 WebSocket 实时推送。
- 支持 Webhook / 邮件等分发配置。
- 支持工作流状态更新与误报反馈相关后端能力。
- 支持告警录像与多轨回放 API。

### 2.4 模型与基线管理

- 支持基线采集与版本化管理。
- 支持训练作业创建、确认、查询。
- 支持模型列表、激活、回滚、批量推理。
- 支持模型发布阶段管理相关后端能力。

### 2.5 Dashboard

当前前端主视图要求如下：

- 总览页：展示系统状态、摄像头状态和近期告警。
- 摄像头页：展示摄像头列表、控制和详情入口。
- 摄像头详情页：展示单路摄像头细节。
- 告警页：展示告警列表与工作流操作。
- 模型页：通过标签页聚合基线、训练、模型、对比、标注和阈值预览。
- 系统页：通过标签页聚合系统概览、配置管理、用户、审计、降级历史、备份与音频告警等能力。

### 2.6 运维能力

- 支持认证中间件、会话、限流、安全头。
- 支持配置重载、数据库备份、存储清理。
- 支持推理记录、审计日志和运行指标。

## 3. 当前可选但非默认能力

以下能力已在代码或配置中出现，但默认不作为系统主路径：

- 开放词汇分类器。
- 分割器。
- 跨摄像头关联。
- 自动重训练。

文档、演示和对外介绍应明确标注这些能力为“可选”或“实验性接入”，避免误导为默认启用。

## 4. 非功能要求

### 4.1 稳定性

- 单路摄像头失败不应导致全局停止。
- 视频流路径不可阻塞普通 API 请求。
- 中间件实现必须兼容长连接与流式响应。

### 4.2 可运维性

- 告警、任务、训练作业、用户行为需要可追溯。
- 关键运行信息需要通过 API、日志或前端页面可见。
- 目录结构应允许长期保存基线、模型、录像和备份。

### 4.3 可配置性

- 主要运行参数通过 YAML 配置。
- 默认配置必须可直接启动本地样例环境。
- 新增配置项时需要同时更新 schema 和默认值。

### 4.4 前后端边界清晰

- 后端提供 API 与状态模型。
- 前端只对已存在的 API 能力做交互封装。
- 不再把“有 API”误写成“已有独立页面”。

## 5. 当前不再使用的表达方式

以下内容不再作为当前需求文档的有效写法：

- 未经验证的精确模块数量、测试数量、页面数量。
- 将关闭状态的能力描述成默认主流程。
- 将后端 API 能力直接等价为前端独立页面。
- 将历史愿景清单直接当作当前交付范围。
